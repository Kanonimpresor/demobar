# DemoBar — e107 CMS Plugin

> Persistent navigation bar for multisite demo environments, inspired by [BootstrapMade](https://bootstrapmade.com/demo/Arsha/). Designed to showcase landing pages, themes and plugins from a single e107 installation powered by the [multisite](https://github.com/CaMer0n/multisite) plugin.

[![e107 CMS](https://img.shields.io/badge/e107-v2.3.3+-blue)](https://e107.org)
[![PHP](https://img.shields.io/badge/PHP-8.0+-purple)](https://php.net)
[![License](https://img.shields.io/badge/License-GPL--3.0-green)](LICENSE)

## Overview

DemoBar injects a fixed top bar across all sites within a multisite e107 installation. It reads the site catalog from `multisite.json` (generated by CaMer0n's multisite plugin) and displays navigation between active demos, status indicators, and call-to-action buttons — all without modifying e107 core files.

### Architecture

```
agency-site.com/                ← Main site (agency portfolio)
agency-site.com/demos/          ← Visual catalog page (auto-generated grid)
agency-site.com/demo-arsha/     ← Demo 1 (with floating demobar)
agency-site.com/demo-saas/      ← Demo 2 (with floating demobar)
agency-site.com/demo-plugin/    ← Demo 3 (with floating demobar)
```

### How It Works

1. **e_header.php** hook injects the bar CSS/JS on every frontend page
2. Reads `multisite.json` from `e_SYSTEM_BASE` to build the demo navigation
3. Identifies the current demo via `e_MULTISITE_MATCH` constant
4. Renders a responsive fixed-top bar with Bootstrap 5
5. Admin can configure bar behavior, colors and CTA links from admin panel
6. Optional cron job resets demo databases to a clean snapshot

## Requirements

| Requirement | Version |
|---|---|
| e107 CMS | ≥ 2.3.3 |

| [multisite plugin](https://github.com/CaMer0n/multisite) | Installed and configured |
| MySQL / MariaDB | ≥ 5.7 / ≥ 10.3 |

## Installation

1. Copy the `demobar/` folder to `e107_plugins/`
2. Go to **Admin → Plugin Manager** and install "DemoBar"
3. Ensure the [multisite plugin](https://github.com/CaMer0n/multisite) is installed and `multisite.json` exists in `e107_system/`
4. Configure the bar from **Admin → DemoBar → Settings**

## Features

### v1.0.1 — Bug Fixes & User Guide (Current)

- [x] **CSS v2.0 rewrite** — Completely rewritten demobar.css with flex layout and responsive breakpoints
- [x] **Cross-database preferences** — `loadPrefs()` reads brand/CTA settings from the main site DB, ensuring consistency across all demos
- [x] **Smart navigation** — Arrows only on demo pages; main site shows a "▶ View Demos" button instead
- [x] **Admin-only button** — Admin button restricted via `check_class(e_UC_ADMIN)`
- [x] **User Guide** — 5-tab admin guide (Overview, Configuration, Usage, Multisite, Troubleshooting) using template + shortcodes + CSS separation of concerns
- [x] **Keyboard shortcut** — `Ctrl+Shift+D` to toggle bar visibility (state saved in localStorage)

### v1.0.0 — MVP

- [x] **Persistent navigation bar** — Fixed top bar injected via `e_header.php` hook
- [x] **Auto-discovery of demos** — Reads active sites from `multisite.json`
- [x] **Current demo indicator** — Highlights which demo the visitor is viewing
- [x] **CTA buttons** — Configurable "View Source" (GitHub) and "Contact" links
- [x] **Previous/Next navigation** — Browse between demos sequentially
- [x] **Hide bar toggle** — Visitors can collapse the bar (stored in localStorage)

## Configuration Options

| Preference | Type | Default | Description |
|---|---|---|---|
| `demobar_active` | boolean | `1` | Enable/disable the bar globally |
| `demobar_brand_text` | text | `"Demos"` | Brand text shown on the left side |
| `demobar_brand_url` | url | `""` | URL when clicking the brand |
| `demobar_bg_color` | text | `"#343a40"` | Bar background color |
| `demobar_text_color` | text | `"#ffffff"` | Bar text color |
| `demobar_cta_source_url` | url | `""` | "View Source" button URL (GitHub) |
| `demobar_cta_source_text` | text | `"Source Code"` | "View Source" button label |
| `demobar_cta_contact_url` | url | `""` | "Contact" / "Use this" button URL |
| `demobar_cta_contact_text` | text | `"Contact Us"` | Contact button label |
| `demobar_show_prevnext` | boolean | `1` | Show Previous/Next demo arrows |
| `demobar_show_hide_toggle` | boolean | `1` | Allow visitors to hide the bar |
| `demobar_admin_ip_whitelist` | textarea | `""` | IPs allowed to access demo admin areas |
| `demobar_position` | dropdown | `"top"` | Bar position: top or bottom |

### Build & Test

```bash
cd e107_tests
composer update
./vendor/bin/codecept run unit tests/unit/plugins/demobar/
```

### Coding Standards

- PHP 8.0+ syntax, 4-space indentation
- All services via `e107::` static API — never instantiate handlers directly
- All strings via `LAN_DEMOBAR_*` constants — no hardcoded text
- Code comments in Spanish, code and constants in English
- Every PHP file starts with `if (!defined('e107_INIT')) { exit; }`
